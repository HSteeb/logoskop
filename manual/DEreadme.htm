Logoskop-Geschichte

Was ist Logoskop?

Logoskop (griech. Wort-Betrachter) zeigt Das Wort - zwei sich ergänzende Bibelworte für jeden Tag des Jahres, herausgegeben von Projekt Bibel 2.0 http://bible2.net/.

Autor: Helmut Steeb http://www.jsteeb.de/. Ich danke meinem Herrn Jesus Christus, der mir ermöglicht, Logoskop zur Verfügung zu stellen.

Für Schnelleinsteiger

Nach der Installation

  • zeigt Logoskop Das Wort für den aktuellen Tag an
  • ... für Deutsch (falls Ihr Betriebssystem deutsch eingestellt ist) oder Englisch
  • ... aus den im Installationspaket enthaltenen Das Wort-Dateien
  • ... bis das Jahr vergangen ist - dann brauchen Sie Dateien für das kommende Jahr (verwenden Sie Menü Datei-Das Wort aktualisieren).

Features

  • Kopieren Sie Das Wort in die Windows-Zwischenablage (Menü Bearbeiten-Kopieren).
  • Lassen Sie Logoskop bei der Installation in den Autostart-Ordner eintragen, so dass es bei jedem Start von Windows automatisch Das Wort anzeigt
  • betrachten Sie Das Wort in verschiedenen Bibelausgaben und Sprachen. Wählen Sie die Bibelausgaben in den Auswahllisten im Toolbar oder in Menü Datei-"Das Wort organisieren".
  • fügen Sie zusätzliche Texte (*.twd) für andere Sprachen oder für kommende Jahre einfach hinzu (Menü Datei-"Das Wort organisieren")
  • schalten Sie die Sprache der Programmoberfläche (Menüs und Meldetexte...) um (Menü Bearbeiten-Einstellungen; die neue Sprache wird nach dem nächsten Start von Logoskop verwendet).
  • Logoskop kann anzeigen, welche Bibeltexte verwendet wurden: Menü Ansicht | Copyright. Um wieder Das Wort anzuzeigen, wählen Sie erneut Menü Ansicht | Copyright (oder einen anderen Tag oder eine andere Sprache).
  • schlagen Sie übers Internet weitere ergänzende Bibelworte vor (Menü Bearbeiten | "Ergänzende Bibelworte vorschlagen").
  • Logoskop kann eine E-Mail-Signatur für Das Wort erzeugen

Installation

  • Systemanforderungen: Windows 9x/NT/2000/XP/Vista/Windows 7 (auch unter Linux, getestet mit Wine 1.0.1)
  • Führen Sie das Installationsprogramm aus. Anschließend wird Logoskop automatisch gestartet.
  • Wenn Sie bei der Installation Probleme mit der Schreibberechtigung im Installationsordner C:\Programme bekommen, können Sie Logoskop einfach in einen anderen Ordner installieren (z.B. "D:\Logoskop").
  • Das Installationsprogramm enthält Texte für das kommende Jahr. Nach der Installation im vorigen Jahr weist Logoskop Sie darauf hin, dass für den Rest des Jahres als Behelf Das Wort des kommenden Jahres angezeigt wird. Sie können auch den Text des laufenden Jahres einfach aus dem Internet herunterladen (Menü Datei | "Das Wort aktualisieren")
  • Falls Das Wort nicht richtig angezeigt wird, versuchen Sie es im Menü Bearbeiten | Layout mit einer anderen Schriftart.

Copyright

Für die Bibeltexte: Menü Ansicht | "Copyright".

E-Mail-Signatur für Das Wort erzeugen

Funktionsweise

In gängigen E-Mail-Programmen können Sie eine "Signatur" definieren. Wenn Sie eine neue E-Mail erstellen, fügt das E-Mail-Programm die Signatur automatisch ans Ende des noch leeren Textes, bevor Sie ihn bearbeiten. Typischerweise enthält die Signatur Ihren Namen und Ihre E-Mail-Adresse, wie z.B.

--
Max Mustermann
Max_Mustermann@mmm.org

Wenn Ihr E-Mail-Programm die Signatur in einer Textdatei auf der Festplatte speichert, können Sie Logoskop so konfigurieren, dass es bei jedem Programmstart automatisch den aktuellen Text von Das Wort in diese Datei schreibt. Erstellen Sie danach eine neue E-Mail, so enthält sie bereits Das Wort als Signatur.

Logoskop schreibt Das Wort als einfachen Text mit Zeilenumbrüchen, ohne Angaben zu Farben oder Schriftarten. Wenn Sie üblicherweise E-Mails im HTML-Format erstellen, wird die von Logoskop geschriebene Signatur nicht wie erwartet aussehen.

E-Mail-Signatur konfigurieren - welche Datei?

Die wichtigste Frage lautet: aus welcher Datei liest Ihr E-Mail-Programm die Signatur neu erstellter E-Mails?

Um dies herauszufinden, können Sie wie folgt vorgehen:

  • kopieren Sie ein (möglichst unverwechselbares) Stück Text aus der Signatur einer neu verfassten E-Mail (notfalls einfach Ihre E-Mail-Adresse).
  • suchen Sie auf Ihrer Festplatte nach einer Datei, die dieses Stück Text enthält. Dies können Sie erledigen mit der Suchfunktion aus Startmenü - Suchen - "Dateien/Ordner" und den Sucheinstellungen:
    DialogfeldTypische Einstellung für die Suche
    Suchen in:C:\
    Name:*.txt
    Untergeordnete Ordner einbeziehenankreuzen!
    Enthaltener Text:das Stück Text
  • wenn eine Datei gefunden wird, prüfen Sie, ob diese tatsächlich Ihre Signatur enthält
  • tragen Sie den vollständigen Pfad zu dieser Datei in den Logoskop-Konfigurationsdialog ein
  • wählen Sie in Logoskop "Signatur jetzt schreiben"
  • erstellen Sie in Ihrem E-Mail-Programm eine neue E-Mail, um zu prüfen, ob Sie die richtige Datei angegeben haben

Verschiedene E-Mail-Programme legen Signaturen an unterschiedlicher Stelle ab. Hier eine unvollständige Liste von E-Mail-Programmen und den Ordnern, in denen Sie Ihre Signatur finden könnten:

E-Mail-Signatur konfigurieren - Vorlage

Die von Logoskop geschriebene Signatur kann neben dem Bibeltext natürlich auch Ihren Namen und Ihre E-Mail-Adresse enthalten. Wie die Signatur aussehen soll, definieren Sie in einer "Vorlage". Die Vorlage kann beliebigen Text enthalten. An den Stellen, an denen Sie die Das Wort vollständig oder einzelne Teile davon platzieren möchten, fügen Sie spezielle Platzhalter ein. Logoskop ersetzt diese Platzhalter dann durch den Text des jeweiligen Tages.

Beispiel:

--
Max Mustermann
Max_Mustermann@mmm.org
--
%TL;

%PAROL1;

%PAROL2;
--

wird z.B. am 2008-11-01 zu

--
Max Mustermann
Max_Mustermann@mmm.org
--
Das Wort für Samstag, 1. November 2008

Die Hirten kehrten wieder um
und priesen und lobten Gott für alles,
was sie gehört und gesehen hatten,
so wie es ihnen gesagt worden war.
Lukas 2,20

Lobt den Herrn, alle seine Werke,
an allen Orten seiner Herrschaft!
Lobe den Herrn, meine Seele!
Psalm 103,22
--

Logoskop erkennt die folgenden Platzhalter:

PlatzhalterBedeutung
%THEWORD;der gesamte Text von Das Wort mit Datumszeile
%TL;die Datumszeile
%PAROL1;die ersten Bibelverse
%PAROL2;die zweiten Bibelverse

HTML-E-Mail

Folgendes Beispiel (von Axel Schlüter 2005-11-28 für w32los) erzeugt eine Signatur für eine HTML-formatierte E-Mail, verwendbar im E-Mail-Programm Thunderbird:

<font face="Tahoma">
<div style="position:relative; left:30px; color:white; margin:30px 30px 30px 0px;
      background-color:#000040; width:280px; padding:30px 20px 20px 30px;">
%TL;<br>
<br>
%PAROL1;<br>
<br>
%PAROL2;
</div>
</font>

Einschränkungen

  • Logoskop schreibt die Platzhalter nur als Text, nicht als HTML (siehe dazu auch unter "SignatureLineSeparator" im Abschnitt INI-Datei)
  • Logoskop schreibt die Signaturdatei bei jedem Programmstart - nicht seltener, nicht öfter
  • Logoskop verwendet für die Signaturdatei den Text in der ersten beim Programmstart angezeigten Sprache
  • Logoskop unterstützt zwei Kodierungen für die Signatur: die auf Ihrem Betriebssystem eingestellte Kodierung, und UTF-8 (dies steuern Sie über "als UTF-8 schreiben" im Dialog "E-Mail-Signatur konfigurieren")
  • wenn Logoskop den Text in der auf Ihrem Betriebssystem eingestellten Kodierung schreibt, können manche Zeichen des angezeigten Textes möglicherweise nicht wiedergegeben werden. Diese werden dann in der Signaturdatei typischerweise durch "?" ersetzt (prüfen Sie dies einfach durch die Schaltfläche "Signatur jetzt schreiben" im Konfigurationsdialog). Der chinesische Text wird auf einem deutschen Betriebssystem zu Vielen Fragezeichen führen.
  • "als UTF-8 schreiben" ist möglicherweise für Ihr E-Mail-Programm nützlich. Allerdings kann die Vorschau unten im Dialog von Logoskop für UTF-8 nur Ascii-Zeichen (A-Z...) korrekt anzeigen.

Technische Einzelheiten

Speicherort der Das Wort-Dateien

Logoskop speichert die Texte für Das Wort (.twd-Dateien) in einem speziellen Windows-Systemordner, in dem Anwendungsdaten abgelegt werden können:

  • Windows XP: C:\Dokumente und Einstellungen\IHRNAME\Anwendungsdaten\bible2.net\
  • Windows Vista: C:\Users\IHRNAME\AppData\Roaming\bible2.net\
  • Windows NT: C:\Windows\Profile\IHRNAME\Anwendungsdaten\bible2.net\

Dateiformat, Namenskonventionen und Datumsauswahl

Logoskop zeigt Texte im .twd XML-Format an, welche einer Document Type Definition (DTD) namens thewordfile entsprechen (diese bekommen Sie in bible2.net, Sie benötigen sie aber nicht zur Anzeige des Textes).

Logoskop verwendet die Standard-Namenskonvention z.B. "de_Schlachter2000_2009.twd", d.h.

  • der Sprachcode (dieser muss mit dem Attribut xml:lang in der Datei übereinstimmen)
  • der Name des verwendeten Bibeltextes
  • die vierziffrige Jahreszahl
  • die Dateinamens-Endung "twd" (the word)

Wenn Logoskop das Wort des aktuellen Tages anzeigt, erkennt es jede Datumsänderung und zeigt den danach aktuellen Text an (am 31. Dezember wechselt es zu einer anderen Datei - unter Verwendung der Standard-Namenskonvention).

Wenn Sie den Text eines anderen Jahres anzeigen lassen, liest Logoskop die Liste der für dieses Jahr verfügbaren Texte neu. Ist kein Text verfügbar, wird eine Fehlermeldung angezeigt.

Start nur einmal am Tag

Im Optionen-Dialog können Sie einstellen, dass Logoskop nur einmal pro Tag per Autostart (d.h. beim Start von Windows) gestartet werden soll. Wenn Sie dies eingestellt haben, bricht das Programm bei nachfolgenden Starts am selben Tag ab, bevor das Programmfenster sichtbar wird. Beachten Sie: diese Einstellung sowie die Angabe, wann Logoskop zuletzt gestartet wurde, werden in der INI-Datei abgelegt. Falls Sie verschiedene INI-Dateien verwenden, ist die "Start nur einmal am Tag"-Einstellung für jede INI-Datei separat wirksam. Wenn Sie Logoskop direkt über das Programme-Menü oder im Explorer starten, wird der Schalter nicht beachtet.

Kopieren in die Zwischenablage

Logoskop kopiert den Text in zwei verschiedenen "Formaten" in die Zwischenablage: im üblichen "Text"-Format und im "Unicode"-Format.

  • "Unicode"-Format: dieses Zwischenablageformat kann Text in (ziemlich) allen Sprachen aufnehmen.
  • "Text"-Format: dieses Zwischenablageformat kann nur Text einer bestimmten "Codepage" aufnehmen, z.B. der westeuropäischen oder der russischen Codepage. In Ihrem Windows-System ist eine bestimmte Codepage voreingestellt.
    Logoskop kann nur Text der eingestellten Codepage in die "Text"-Zwischenablage kopieren.
    Wenn Logoskop einen Text einer "fremden" Sprache anzeigt (z.B. Russisch auf einem westeuropäischen System), dann kann das "Text"-Format der Zwischenablage diesen Text nicht aufnehmen; es interpretiert die Zeichen entweder fälschlicherweise als Zeichen der eingestellten Codepage, oder es ersetzt die unbekannten Zeichen z.B. durch Fragezeichen.

Manche Anwendungsprogramme holen Text automatisch im "Unicode"-Format aus der Zwischenablage ab (z.B. Word 2000), diese können also den Text beliebiger Sprachen korrekt verwenden.

Auf Windows 9.x-Systemen verwenden manche Anwendungsprogramme nur das "Text"-Format der Zwischenablage. In solche Anwendungsprogramme können Sie Texte, die in einer anderen Sprache vorliegen, nicht verlustfrei einfügen.

Arabischer Text wird gemäß Unicode in der "logischen Anordnung" in die Zwischenablage übertragen.

Download

Mit Logoskop können Sie Das Wort-Dateien aus dem Internet herunterladen.

  • im Dialog Das Wort aktualisieren (Herunterladen der Texte für die gerade angezeigten Bibelausgaben für das laufende oder kommende Jahr), und
  • im Dialog Das Wort organisieren (Herunterladen mehrerer Texte auf einmal; für jedes Jahr, für das Das Wort auf bible2.net verfügbar ist).

Um Das Wort-Dateien herunterladen zu können, benötigen Sie eine funktionierende Internet-Verbindung. In einer Firma können Sie möglicherweise eine Internet-Verbindung nur über einen Proxy oder Firewall aufbauen. Logoskop unterstützt drei Verwendungsweisen von Proxies:

  • Proxy-Einstellungen des Internet Explorer verwenden. Diese können Sie z.B. in der Windows-Systemsteuerung (Internetoptionen) bearbeiten.
  • keinen Proxy verwenden. Dies ist die richtige Wahl, wenn Sie Logoskop zuhause über ein Modem betreiben.
  • die Proxy-Adresse manuell angeben, z.B. "www-proxy". Sie können zusätzlich den Anschluss angeben, z.B. "www-proxy:8080" (trennen Sie Adresse und Anschluss durch ":"; der voreingestellte Anschluss ist 80). Fragen Sie bitte Ihren Netzwerkadministrator, wenn Sie die Proxy-Adresse nicht wissen, oder sehen Sie in den Einstellungen des Browsers nach (z.B. Firefox 3: Extras-Einstellungen-Erweitert-Netzwerk)

Wenn Ihr Proxy für eine Verbindung ins Internet einen Benutzernamen und ein Kennwort erfordert, fragt Logoskop diese ab, sobald Sie einen Download versuchen. Aus Sicherheitsgründen speichert Logoskop weder Ihren Benutzernamen noch das Kennwort.

Beim Download kommuniziert Logoskop mit einem HTTP-Dienst auf bible2.net. Logoskop übermittelt dem Dienst nur Daten, die für die HTTP-Kommunikation erforderlich sind.

Wenn Sie Schwierigkeiten haben, mit Logoskop eine Verbindung ins Internet aufzubauen, und Logoskop zu früh mit einer Fehlermeldung abbricht, können Sie versuchen, die Zeitverzögerung zu erhöhen (siehe "Timeout" im Abschnitt INI-Datei).

Achtung: manche Internet-Provider lenken in jeder Internet-Verbindung die erste eingegebene Webadresse (d.h. die erste HTTP-Anfrage) auf die Startseite des Providers um. Logoskop lädt die Das Wort-Dateien per HTTP herunter. Wenn Sie also in einer Internet-Verbindung als erste Aktion Das Wort-Dateien mit Logoskop herunterladen, dann wird auch die erste HTTP-Anforderung, die Logoskop absendet, vom Provider umgeleitet und bekommt als Antwort die HTML-Startseite des Providers anstelle einer Das Wort-Datei. Sollten Sie also beim Herunterladen der Das Wort-Dateien unerklärliche Fehlermeldungen bekommen, dann könnte es helfen, wenn Sie zuerst im Internet-Browser irgendeine Webseite besuchen und erst dann - in derselben Internet-Verbindung - in Logoskop eine Das Wort-Datei herunterladen.

Beschreibung des Datums-Eingabefeldes

Logoskop verwendet ein Standard-Windows-Eingabefeld, um das Datum des anzuzeigenden Textes auszuwählen. In alten Windows 95-Versionen kann dieses Eingabefeld fehlen, in diesem Fall bleibt der entsprechende Bereich des Toolbar leer (technisch ausgedrückt: Sie benötigen Version 4.70 oder höher der Comctl32.dll, welche mit Microsoft Internet Explorer Version 3.x oder höher geliefert wird).

Das Datums-Eingabefeld verarbeitet die folgenden Eingaben:

Art der EingabeBeschreibung
Pfeiltasten Sie können mit den Pfeiltasten links/rechts durch den Eingabebereich wandern.
Die Tasten auf/ab ändern den Wert im aktuellen Feld schrittweise.
Ende und Pos1 setzen den Wert des aktuellen Feldes auf den maximal bzw. minimal möglichen Wert
Funktionstasten F2 aktiviert den Bearbeitungsmodus.
F4 läßt eine Monatsliste erscheinen (Alt-Pfeil ab hat denselben Effekt).
Ziffern Das Eingabefeld verarbeitet die Eingabe von Zahlen in Bereichen von jeweils zwei Ziffern.
Plus und Minus vergrößern bzw. verkleinern den Wert im aktuellen Bereich

Kommandozeilen-Parameter

Aufruf von Logoskop: 

  Logoskop [-i INIDATEI][-autostart]

Logoskop verarbeitet als optionalen Parameter: der Pfad zur ini-Datei. Hiermit können Sie Logoskop mehrfach starten und bei jedem Start eine andere ini-Datei angeben. Dann können Sie die verschiedenen Programmfenster an unterschiedlichen Stellen der Arbeitsoberfläche platzieren, Das Wort in unterschiedlichen Sprachen anzeigen, andere Farben und Zeichensätze wählen usw.

Zum Parameter "-autostart" siehe Start nur einmal am Tag.

INI-Datei

Logoskop speichert Einstellungen in einer Datei Logoskop.ini im Verzeichnis Anwendungsdaten\bible2.net (der Ordner "Anwendungsdaten" liegt je nach Windows-Version an einer anderen Stelle).

Spezielle Werte (nicht über Dialoge änderbar):

[General]
Timeout=15000
Wartezeit für Internet-Verbindungen, in 1/1000 Sekunden. Die Voreinstellung beträgt 15000 (also 15 Sekunden).
[General]
SignatureTemplate=PfadZurDatei
zeigt auf eine Datei, in der Logoskop den im Dialog "E-Mail-Signatur konfigurieren" definierten Vorlagentext speichert
[General]
SignatureLineSeparator=<br/>
definiert anstelle des einfachen Zeilenumbruches z.B. den HTML-Befehl "<br/>" als Zeilentrenner. Die Zeichenfolgen "\t", "\r" und "\n" (jeweils zwei Zeichen: Backslash gefolgt von einem Buchstaben) werden durch das entsprechende Steuerzeichen ersetzt.
[General]
LogfileLevel=3
aktiviert die Protokollierung der Programmaktionen in eine Logdatei (Voreinstellung: deaktiviert). Gültige Werte sind: 0 = keine Protokollierung, 1 = nur Fehlermeldungen, 2 = ... zusätzlich informative Meldungen, 3 (und alle anderen Werte) = zusätzlich Statusmeldungen
[General]
Logfile=c:\temp\Logoskop
setzt den Namen der Logdatei (Voreinstellung: "Logoskop.log" im selben Verzeichnis wie Logoskop.exe). Im Optionendialog können Sie zwischen "ein" (= 3) und "aus" (= 0) wählen.
[General]
ParseStrict=1
bewirkt, dass Logoskop eine Das Wort-Datei ablehnt, die unerwartete XML-Elemente enthält (Voreinstellung: 0)
[General]
UseUSP10=0
Tragen Sie ein UseUSP10=0, um nicht den "Microsoft Uniscribe Processor" (USP10.DLL) für die Anzeige zu verwenden, sondern stattdessen einen internen Algorithmus, basierend auf FriBidi (Open Source). Diesen Wert können Sie auch über den Optionendialog ändern. Falls Logoskop die USP10.DLL nicht laden oder verwenden kann (z.B. weil Sie Logoskop mit Wine unter Linux gestartet haben), schaltet es UseUSP10 automatisch aus.
[General]
SetMonthCalFont=0
sollten bei Ihnen Probleme mit der Schriftart des Datums-Eingabefeldes auftreten, könnte diese Einstellung helfen. Sie bewirkt, dass die Schriftart des Eingabefeldes beim Programmstart nicht auf die Schriftart des Toolbar gesetzt wird (Voreinstellung: 1).
