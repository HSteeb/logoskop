What is Logoskop?

Logoskop (greek word viewer) displays The Word - two related bible quotations for each day of the year, published by Project Bible 2.0 http://bible2.net/.

Author: Helmut Steeb http://www.jsteeb.de/. I am grateful to my Lord Jesus Christ who makes me able to provide Logoskop.

Simple usage

After installation,

  • Logoskop automatically displays The Word for the current day
  • ... for your language (guessing from your operating system settings), or English,
  • ... from the respective The Word files contained in the installation package
  • ... until the year is over - then you need files for the next year (you may use menu File | Actualize The Word).

Features

  • copy The Word to the Windows clipboard (menu Edit | Copy).
  • put the application into the startup folder (during the installation)
  • select multiple bible editions in different languages using the comboboxes in the toolbar or menu File | Organize The Word.
  • install additional The Word texts (*.twd) for other languages or for years to come (menu File | Organize The Word)
  • change the language of the user interface (i.e. menus and message texts...) in menu Edit | Options. The new language is used after a restart of Logoskop.
  • Logoskop can display information about the bible texts used using menu View | Copyright. To display The Word again, again toggle menu View | Copyright or select another day or language.
  • suggest additional related bible quotations over the Internet (menu Edit | "Suggest related bible quotations").
  • configure Logoskop to create email signatures

Installation

  • System requirements: Windows 9x/NT/2000/XP/Vista/Windows 7 (also under Linux, tested with Wine 1.0.1).
  • Run the installer. Finally, Logoskop will start automatically.
  • If during installation you experience problems with write permissions in the folder e.g. "C:\Program files", try to install Logoskop into a different folder, e.g. "D:\Logoskop".
  • The installer contains the text files for the year to come. After installation in the previous year, Logoskop tells you that - for the rest of the year - the texts for the year to come will be displayed. You may conveniently download the texts for the current year (using menu File | "Actualize The Word")
  • If The Word is not displayed correctly, try menu Edit | Layout and select a different font.

Create an email signature that contains The Word

How it works

In popular email programs you may define a "signature". Any time you create a new email, your email program adds the signature automatically at the end of the new text, before you edit the text. Typically, the signature contains your name and your email address, like

--
John Doe
john@doe.org

If your email program stores the signature in a text file on your hard disk, you may configure Logoskop to write The Word into this file (at the time Logoskop starts). Any time you create a new email, then, the email will already contain The Word as a signature.

Logoskop writes The Word as simple text with line breaks, without special colour or font information. If you use to write your emails in HTML format, the signature written by Logoskop will probably not look as you expect.

Configuring the email signature - which text file?

The most important question for using this Logoskop feature is: which is the signature file that your email program reads for new emails?

To answer this question, you may proceed as follows:

  • copy some text from the signature of a new email to the Windows clipboard. The text should be unique, you may just try your email address.
  • on your hard disk, search for a file that contains this text. You may use: Start menu - Search - "Files/Folders", with the search options:
    Dialog itemTypical search option
    Search in:C:\
    Name:*.txt
    Include subfolderscheck this!
    Text contained:your text
  • if you find a file, please check whether it actually contains the signature
  • enter the complete path to this file into the Logoskop configuration dialog
  • in Logoskop, press "Write signature now"
  • create a new email in your email program - if the signature is there, you're done!

Different email programs store their signature at different locations on your hard disk. Here is an incomplete list of email programs and the folders where you may find the signature file:

Mozilla 1.4Menu Edit | Mail & Newsgroup Account Settings | path in "Append this signature"
MS Outlook ExpressMenu Extras | Options | Signatures | Edit Signatures

Configuring the email signature - the Template

The signature that Logoskop writes should contain not only The Word but also your name and email address, of course. You define the contents of the signature in a "template". This template may contain any text. At the places where you want to appear the complete The Word text or parts of it, you enter some special placeholders. Logoskop will replace these placeholders by the text of the respective day.

Example:

--
John Doe
john@doe.org
--
%TL;

%PAROL1;

%PAROL2;
--

will be expanded e.g. at 2008-11-01 to

--
John Doe
john@doe.org
--
The Word for Saturday, November 1, 2008

The shepherds returned,
glorifying and praising God
for all they had heard and seen,
as it had been told them.
Luke 2:20 (ESV)

Bless the Lord, all his works,
in all places of his dominion.
Bless the Lord, O my soul!
Psalm 103:22 (ESV)
--

Logoskop detects the placeholders:

PlaceholderMeaning
%THEWORD;the complete text of The Word including date line
%TL;the date line
%PAROL1;the first bible verses
%PAROL2;the second bible verses

HTML Email

Here is an example (donated by Axel Schlüter 2005-11-28 for w32los) to produce a signature usable in an HTML formatted email with the Thunderbird mail program:

<font face="Tahoma">
<div style="position:relative; left:30px; color:white; margin:30px 30px 30px 0px;
      background-color:#000040; width:280px; padding:30px 20px 20px 30px;">
%TL;<br>
<br>
%PAROL1;<br>
<br>
%PAROL2;
</div>
</font>

Restrictions

  • Logoskop writes the placeholders only as text, not as HTML (see also in "SignatureLineSeparator" in the chapter INI File)
  • Logoskop writes the signature file at every start
  • Logoskop uses The Word in the first language that is displayed at program start
  • Logoskop supports two encodings for the signature: the encoding that is used in your operating system, and UTF-8 (toggled by the "file is in UTF-8" checkbox in the "Configure email signature" dialog)
  • if Logoskop writes the text in the encoding that is used in your operating system, and you write The Word in a language with characters not supported by that encoding, the characters will typically be replaced by a "?" (just check this using the button "Write to file now" in the configuration dialog). On a Western operating system, e.g. Chinese text will produce Lots Of Question Marks.
  • "file is in UTF-8" may produce the right result for your email application, but for UTF-8, the preview at the bottom of the Logoskop dialog will only display Ascii characters (A-Z...) correctly.

Technical details

Location of The Word files

Logoskop organizes the texts for The Word (.twd files) in a special Windows folder where applications may store their data, like:

  • Windows XP: C:\Documents and Settings\YOURNAME\Application Data\bible2.net\
  • Windows Vista: C:\Users\YOURNAME\AppData\Roaming\bible2.net\
  • Windows NT: C:\Windows\Profiles\YOURNAME\Application data\bible2.net\

File naming, language and date selection

Logoskop displays files in .twd XML format that conform to a document type definition called thewordfile (which you may download from bible2.net, but you do not need it to display the text).

Logoskop assumes the standard file naming convention e.g. "de_Schlachter2000_2009.twd", i.e.

  • the "language code" (may but need not be the same as in the xml:lang attribute within the file)
  • the name of the bible used
  • the four digit year number
  • the extension "twd" (the word)

When it displays the current day, Logoskop recognizes any changes of the date and re-displays the text for the now current date (on dec 31, it might even switch to another file using the naming convention).

When you switch to another year, Logoskop re-reads the contents of the application directory to see which languages are available. If there is no language available in the new year, an error message will be displayed.

Start only once per day

In the options dialog you may choose to start Logoskop via startup folder only once per day. If this is enabled, Logoskop simply exits before the application window appears in case you start it more than once. Note that this setting as well as the fact when Logoskop has been started the last time is stored in the INI file - if you use different INI files, the "start only once" applies to each INI file separately. The option is not checked if you start Logoskop directly from the applications menu or in Windows Explorer.

Copying to the clipboard

Logoskop copies the text into the clipboard using two different "clipboard formats": the regular "Text" format and the "Unicode" format.

  • "Unicode" format: this clipboard format can hold texts in (almost) any language.
  • "Text" format: this clipboard format can hold only text of one special "codepage", e.g. the West European, or the Russian codepage. Your system uses a "default codepage" (depending e.g. on your version of Windows). Logoskop can display text without regard to the default codepage, but the "Text" format of the clipboard can only hold text in the default codepage.
    If Logoskop displays a "foreign" language (e.g. Russian on a West European Windows), the "Text" format of the clipboard cannot hold these data, it either mis-interpretes them as characters of its current codepage, or replaces them by an "unknown character indicator" (typically a question mark).

Some applications automatically fetch text from the clipboard in "Unicode" format (e.g. current Word versions), these will work appropriately.

On Windows 9.x, applications typically use the "Text" format only, you may not be able to paste foreign language text into such an application.

Arabic text will be copied in "logical order" (i.e. in the order of writing from left to right) without shaping and ligatures.

Download

In Logoskop, you may download The Word texts using

  • the dialog Actualize The Word (allows download of text for currently displayed bible editions and for the current year or the year to come), and
  • the dialog Organize The Word (allows download of several texts at once, for any year for which project Bible 2.0 has got texts).

To download texts, you need a working connection to the Internet. If you're in a company, you probably cannot access the Internet directly but only via a proxy or firewall. There are three ways for Logoskop to handle a proxy:

  • use the proxy settings from Internet Explorer. You can modify these e.g. in the Windows control panel (Internet options).
  • do not use any proxy. This is ok e.g. if you're running Logoskop at home with a modem.
  • set proxy address manually. The address might look like "www-proxy". You may also specify the port for accessing the proxy, like "www-proxy:8080" (i.e. address and port are separated by ":"; port 80 is the default). Ask your network administrator if you don't know the address, or have a look e.g. in Firefox settings (Firefox 3: Tool-Preferences-Extended-Network, or the like)

If your proxy needs a username and password to connect to the Internet, Logoskop will query these as soon as you try to download data. For security reasons, Logoskop does not remember your username or password.

For downloading files, Logoskop communicates with a HTTP service running at bible2.net. Logoskop does not transmit information to the service besides data needed for HTTP communication.

If you have problems establishing a connection, and Logoskop stops trying too soon, you may increase its timeout value (see "Timeout" in the Ini file chapter).

Note: some Internet providers redirect the first URL you enter (the first HTTP request) to their own site, for each of your Internet connections. Logoskop uses HTTP to download the The Word. Therefore, if the first action in your Internet connection is to download The Word using Logoskop, then the first HTTP request of Logoskop gets redirected, too, and receives an HTML page instead of a The Word file! If you experience mysterious failure when you try to download The Word, you may try to visit any web page in your browser first. Then - in the same Internet connection - try downloading a The Word file in Logoskop.

Date and time picker control behaviour

Logoskop uses a Date and time picker control to change the date the watchword is displayed for. Some Windows 95 versions may not have this Date and time picker control, in this case the respective area of the toolbar remains empty (technically speaking: you need version 4.70 and later of Comctl32.dll, which comes with Microsoft Internet Explorer 3.x or greater).

The Date and time picker control accepts input as follows:

Input categoryDescription
Arrow Keys The user can press the LEFT ARROW or RIGHT ARROW keys to move through the control.
The UP ARROW and DOWN ARROW keys change values in the current field incrementally.
End and Home change the value within the current field to its upper and lower limits, respectively.
Function Keys The F2 key activates edit mode.
The F4 key causes the control to display a drop-down month calendar control (pressing ALT+DOWN ARROW does this as well).
Numbers The control accepts numeric input in two-character segments.
Plus and Minus increment and decrement the value in the current field

Command line arguments

Usage: 

  Logoskop [-i INIFILE][-autostart]

Logoskop accepts as optional argument the path of the ini file. You might start Logoskop several times, each time specifying a different ini file. Then you can place these Logoskop instances at different places on your desktop, select different languages and colours within them etc.

For the parameter "-autostart", see Start only once per day.

INI file

Logoskop stores settings in the file Logoskop.ini in the directory AppData\bible2.net (the location of the folder "ApplicationData" depends on your version of Windows).

Special settings (not controlled by any dialog):

[General]
Timeout=15000
this is the timeout used for Internet connections, in 1/1000 seconds. It defaults to 15000 (i.e. 15 seconds).
[General]
SignatureTemplate=PathToFile
denotes a file in that Logoskop stores the template text which you define in the dialog "Configure email signature"
[General]
SignatureLineSeparator=<br/>
defines e.g. the HTML command "<br/>" to be used as line separator, instead of a simple line break. The strings "\t", "\r" und "\n" (two characters, respectively: a backslash followed by a letter) will be replaced by the respective control character.
[General]
LogfileLevel=3
activates the logging of application actions into a logfile (default: 0 = not activated). Valid values are: 0 = no logging, 1 = log only error messages, 2 = ... plus info messages 3 (and all other values) = ... plus internal status messages. In the Options dialog, you select between on (=3) and off (= 0).
[General]
Logfile=c:\temp\Logoskop
sets the name of the logfile (default: "Logoskop.log" in the directory of Logoskop.exe)
[General]
ParseStrict=1
lets Logoskop reject any The Word file that contains unexpected XML elements (default: 0)
[General]
UseUSP10=0
Set UseUSP10=0 to disable use of the Microsoft Uniscribe Processor (USP10.DLL) for displaying texts. An internal processor (based on the open source FriBidi algorithm) will be used instead. You may modify this setting via the Options dialog. If Logoskop fails to load or use the USP10.DLL (e.g. when run under Linux using Wine), it will automatically turn UseUSP10 off.
[General]
SetMonthCalFont=0
if you have problems with the font in the calendar control, you may try adding this setting. Its effect is that the font of the calendar is not initially set to the font of the toolbar (default: 1).
